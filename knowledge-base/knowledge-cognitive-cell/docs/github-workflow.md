# GitHub Workflow

The knowledge base should be maintained through a simple review workflow.

## Standard Flow

```text
Find need
-> Choose template
-> Create Markdown entry
-> Add metadata
-> Commit changes
-> Open pull request
-> Review
-> Merge
-> Update index if needed
```

## Pull Request Description

Each pull request should answer:

1. What knowledge did you add or update?
2. Who is the audience?
3. Which module does it support?
4. Is it ready for use or still draft?
5. Are there related entries?

## Ownership

- Course entries should be reviewed with Curriculum Team.
- Challenge entries should be reviewed with Challenge Team.
- Agent entries should be reviewed with Agent Team.
- Ontology-related tags should be reviewed with Ontology Team.
- Platform integration notes should be reviewed with Platform Team.

Knowledge Team maintains the structure, templates, and quality rules.
