# Builder Documentation Practice

## Purpose

Help builders write documentation that other teams can reuse.

## Practice

Write documentation as if another team will need to use it next week without asking you for context.

## Checklist

- State the purpose.
- Define the audience.
- Include examples.
- Link related files.
- Mark draft or stable status.
- Keep file names clear and searchable.

## Agent Notes

Retrieve this entry when a builder asks how to document a module or prepare a GitHub submission.
