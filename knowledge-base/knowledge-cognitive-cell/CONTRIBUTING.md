# Contributing to NSEAP Knowledge Base

This guide explains how builders should submit knowledge items.

## Contribution Workflow

1. Create or update a branch.
2. Pick the correct folder under `knowledge-base/`.
3. Copy the matching template from `templates/`.
4. Rename the file using lowercase words and hyphens.
5. Fill in the metadata frontmatter.
6. Write clear content with examples where possible.
7. Commit and push your changes.
8. Open a pull request.
9. Ask for review from the Knowledge Team or related module owner.

## File Naming

Use lowercase names with hyphens:

```text
week-1-overview.md
build-your-first-ai-assistant.md
coding-coach-prompt.md
student-faq.md
```

## Branch Naming

Recommended branch names:

```text
knowledge/add-course-note
knowledge/add-challenge-guide
knowledge/add-prompt
knowledge/add-faq
knowledge/update-best-practice
```

## Commit Message Examples

```text
docs: add week 1 overview
docs: add coding coach prompt
docs: update student faq
schema: add prompt schema
guide: add github workflow
```

## Review Checklist

Before submitting, check:

- The file is in the correct folder.
- The metadata fields are complete.
- The title is clear.
- The audience is defined.
- The content has a purpose.
- Related items are linked when possible.
- The entry can be understood by someone outside your team.
