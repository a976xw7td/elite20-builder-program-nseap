# Example: Challenge to Cognitive Cell Case

## Purpose

This example shows how one Builder Challenge can become a Cognitive Cell.

The case uses the Evaluation Agent because it is easy to understand:

```text
A student submits a Challenge repo.
The system needs to review it with a Rubric.
The result should become an evaluation report and reusable knowledge.
```

## What This Example Demonstrates

```text
Situation
-> Context
-> Ontology
-> Workflow
-> Skills
-> Agent
-> Evaluation
-> Knowledge Capture
```

## File Map

- `situation.md`: the real situation behind the Agent
- `context.md`: what information the Agent needs
- `ontology-mapping.md`: how Challenge, Skill, Rubric, and Evaluation connect
- `workflow.md`: the review process
- `skills.md`: reusable Skills extracted from the workflow
- `sample-submission/`: a mock student submission
- `evaluation-report.md`: a sample Evaluation Agent output
- `knowledge-capture.md`: reusable knowledge produced by the review

## Main Lesson

In Elite20 Builder Program / NSEAP, a Challenge is not only homework.

A good Challenge should become a reusable Cognitive Cell with:

- identity
- capability
- interface
- workflow
- evaluation
- memory / knowledge updates
