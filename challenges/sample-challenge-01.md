# Sample Challenge 01: Build a Challenge Submission Repo

## Challenge ID

`C01`

## Challenge Name

Build a Challenge Submission Repo

## Background

Elite20 Phase II needs every student submission to be easy to inspect, review, reuse, and improve.

In Phase I, many artifacts were submitted as files in chat groups. That worked for exploration, but Phase II needs a more stable GitHub-based workflow.

This Challenge asks each Builder to create a clean submission repository for one task.

## Goal

Create a GitHub repository that can serve as a standard Elite20 Challenge submission.

The repository should clearly show:

- what the task is
- what you built
- how AI helped you
- how to run or inspect the result
- what you learned
- what future students can reuse

## Target Users

- Student Builder
- Peer reviewer
- Instructor
- Coding Coach Agent
- Evaluation Agent

## Required Inputs

- this Challenge document
- `rubric-template.md`
- `submission-template.yaml`
- your chosen task or mini-project

## Suggested Process

1. Create a GitHub repository.
2. Add a clear `README.md`.
3. Add your project files or documents.
4. Add `ai-log.md` to record AI collaboration.
5. Add `reflection.md` for AAR.
6. Add `submission.yaml`.
7. Ask Coding Coach Agent to pre-check.
8. Ask one peer to review.
9. Submit for Evaluation Agent review.

## Required Submission

Your repo should include:

```text
README.md
ai-log.md
reflection.md
submission.yaml
src/ or docs/
assets/ optional
```

## README Requirements

The README should answer:

- What did you build?
- Why does it matter?
- How can someone inspect or run it?
- What files should reviewers look at first?
- What is the current limitation?

## AI Log Requirements

The AI log should include:

- AI tools used
- prompts or Skills used
- important iterations
- failed attempts
- hallucinations or mistakes found
- final human decisions

## Reflection Requirements

Use a short AAR format:

```text
Goal:
What happened:
What worked:
What did not work:
What I learned:
What I will improve next:
```

## Rubric

Use `rubric-template.md`.

Recommended weighting:

| Dimension | Points |
|---|---:|
| Task completion | 30 |
| AI collaboration process | 20 |
| GitHub / file structure | 15 |
| Documentation quality | 15 |
| Reflection / AAR | 10 |
| Reusable value | 10 |

## Example Review Question

Could a future student open your repo and understand what to learn from it within 5 minutes?
