# Skills

## Skill 1: Repo Completeness Check

Purpose:

Check whether the submission contains all required files.

Inputs:

- required artifact list
- repository file tree

Outputs:

- missing files
- completeness status
- next action

## Skill 2: README Quality Review

Purpose:

Check whether a future student can understand the submission.

Inputs:

- README
- Challenge goal

Outputs:

- clarity assessment
- missing explanation
- improvement suggestions

## Skill 3: AI Log Review

Purpose:

Check whether the student honestly records AI collaboration.

Inputs:

- AI log
- project result

Outputs:

- AI process quality
- missing process evidence
- hallucination / correction notes

## Skill 4: Rubric Scoring

Purpose:

Score the submission against the Rubric.

Inputs:

- Rubric
- submission artifacts
- evidence

Outputs:

- score breakdown
- supporting notes

## Skill 5: Feedback Generation

Purpose:

Produce concrete revision suggestions.

Inputs:

- score breakdown
- missing requirements
- README review
- AI log review

Outputs:

- strengths
- required fixes
- optional improvements
