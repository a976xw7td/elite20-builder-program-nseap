# Builder Workflow for This MVP

This module follows the Builder Workflow from the Elite20 Builder Program.

## Original Workflow

```text
Find problem
-> Propose solution
-> AI-assisted development
-> GitHub submission
-> Peer Review
-> Agent Review
-> Merge
-> Documentation
-> Release
```

## How This MVP Follows It

| Builder Workflow Step | This Module |
| --- | --- |
| Find problem | A folder of Markdown files is useful, but not enough as a product MVP. |
| Propose solution | Build a lightweight Knowledge Cognitive Cell demo app. |
| AI-assisted development | Use AI to create app files, data model, templates, docs, and review checks. |
| GitHub submission | Submit `nseap-knowledge-base/` to the project repository or GitHub Organization. |
| Peer Review | Ask classmates or module owners to review usability, structure, and clarity. |
| Agent Review | Ask an AI agent to review metadata, schema, links, and demo readiness. |
| Merge | Merge reviewed changes into the main project repository. |
| Documentation | Keep `README.md`, `MVP.md`, `PRESENTATION.md`, and `docs/` updated. |
| Release | Tag or announce MVP v0.1 as the first Knowledge Cognitive Cell release. |

## Current MVP Release Candidate

The current release candidate includes:

- Static demo app in `app/`.
- Knowledge data in `app/knowledge-data.json`.
- Markdown knowledge source in `knowledge-base/`.
- Templates in `templates/`.
- Schemas in `schemas/`.
- Contribution workflow in `CONTRIBUTING.md`.
- Presentation notes in `PRESENTATION.md`.

## Review Checklist

Before GitHub submission:

- The app opens and displays entries.
- The search and category filters work.
- Each entry shows metadata, concepts, skills, relationships, and growth flow.
- JSON files are valid.
- README explains how to run the MVP.
- The demo story matches the Knowledge Team task.
